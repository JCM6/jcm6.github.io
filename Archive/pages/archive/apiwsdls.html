SOAP SourceCredentials

MINDBODY authenticates all incoming API requests to ensure the security of our clients and their data. If the provided SourceCredentials element cannot be authenticated, then an InvalidCredentials status is returned.

In the SOAP body, the SourceName, Password, and SiteID elements need to be provided within the SourceCredentials element.

The example below shows the SourceCredentials element as it would appear in the SOAP body. This element is required for all requests.

<SourceCredentials>
          <SourceName>source_username</SourceName>
          <Password>source_password</Password>
          <SiteIDs>
                    <int>site_id</int>
          <SiteIDs>
<SourceCredentials>
ElementValueDescription
SourceNamesource_usernameSigning up for our Partner Program provides you with an API source name. This can be found by logging into the Partner Portal.
Passwordsource_passwordThis is the key provided to you after signing up for the Partner Program. This can be found by logging into the Partner Portal.
SiteIDssite_idThis is the studio ID associated with the MINDBODY Site you've been granted permission to access.



SOAP UserCredentials

Some of our API calls require a UserCredentials element to ensure the correct level of permissions are available to the requester. The UserCredentials element can also be provided in numerous calls to adjust the level of permission the requester has on the API request. These credentials can affect the available actions and response data.

<UserCredentials>
          <Username>studio_username</Username>
          <Password>studio_password</Password>
          <SiteIDs>
                    <int>site_id</int>
          </SiteIDs>
</UserCredentials>
ElementValueDescription
SourceNamestudio_usernameThe owner, staff, or API default username used to log into the MINDBODY Site. See the description below for information regarding the API default username.
Passwordstudio_passwordThe password associated with the username above.
SiteIDssite_idThis is the studio ID associated with the MINDBODY Site you've been granted permission to access.
UserCredentials in RequestDescription
NoneConsumer mode - If UserCredentials are not passed inside a request then the request will operate in consumer mode when the request is executed.
Staff or OwnerBusiness mode - Only staff or owner level logins may be used within this element. When UserCredentials are passed, the request will operate in business mode when executed.


What are business mode and consumer mode?

MINDBODY has two different user environments: business mode and consumer mode. These environments are determined by the level of permissions granted to the login used to access the site. Business mode can be viewed by logging in with any staff level login and will allow access to business management features. Consumer mode is the client-facing side of the software.

When a set of SourceCredentials are granted access to a client site, a set of UserCredentials are automatically added on the back end of the site. These credentials are based on the SourceName and will use the same password as your SourceCredentials, except with an underscore placed in front of the other characters. These credentials are equivalent to a staff level login.

For example:

SourceCredentials: TestCreds1UserCredentials: _TestCreds1
Source Key: Example123Password: Example123



XMLDetail

Every request contains an element called XMLDetail, which accepts either "Bare," "Basic," or "Full" as a valid input. This element determines how much information is serialized to XML and provided in the response. If a default value is not specified, then most methods will return "Full" information.

XMLDetail InputDescription
BareReturns only a node and an ID. This can be used for simple object existence checks.
BasicReturns a full list of objects with their IDs and descriptions.
FullReturns a full list of objects with their IDs, descriptions and additional information.



Paging

Our API offers paging elements to help restrict the amount of data returned for large calls. PageSize determines how many results are returned per page and CurrentPageIndex determines which page of results is returned.

<PageSize>page_size</PageSize>
<CurrentPageIndex>page_index</CurrentPageIndex>
ElementValueDescription
PageSizepage_sizeThe integer value provided here determines how many results are returned per page. If a page size of "0" is provided, then all results will be returned in a single page. Some calls do not allow for a page size of "0" -- these will return an error message.
CurrentPageIndexpage_indexThe integer value provided here determines which page of results are returned. Pages start at zero and iterate from there. If there is only a single page of results, then the value provided in CurrentPageIndex is ignored.



Namespace

The namespace defines the elements and attributes in the SOAP message. The MINDBODY API uses a namespace definition that you should provide in the SOAP envelope. This should be represented as xmlns="http://clients.mindbodyonline.com/api/0_5"". Below is an example of the namespace definition in an empty request.

<soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/"" xmlns="http://clients.mindbodyonline.com/api/0_5">;
   <soapenv:Header/>
   <soapenv:Body>
   </soapenv:Body>
</soapenv:Envelope>


Character Encoding

All incoming requests must be encoded using UTF-8 (UCS Transformation Format—8-bit). Additionally, all API responses are returned using UTF-8.
 


Structure of Requests

The example below describes the elements needed by the MINDBODY SOAP API in order to process requests.

<soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/"" xmlns="http://clients.mindbodyonline.com/api/0_5">;
   <soapenv:Header/>
   <soapenv:Body>
     <AddClientsToClasses>
         <Request>
            <SourceCredentials>
               <SourceName>api_username</SourceName>
               <Password>api_password</Password>
               <SiteIDs>
                  <int>site_id</int>
               </SiteIDs>
            </SourceCredentials>
            <XMLDetail>Full</XMLDetail>
            <PageSize>page_size</PageSize>
            <CurrentPageIndex>page_index</CurrentPageIndex>
            <ClientIDs>
               <string>client_id</string>
            </ClientIDs>
            <ClassIDs>
               <int>class_id</int>
            </ClassIDs>
         </Request>
      </AddClientsToClasses>
   </soapenv:Body>
</soapenv:Envelope>
